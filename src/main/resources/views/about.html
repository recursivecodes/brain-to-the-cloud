About This Project...

The goal of this project is to determine the relationship between cognitive function and video game performance. In other words, when I'm focused and attentive, do I play better? Or, when I'm distracted, do I play worse? Or, is there no connection at all? Can I even test this? Who knows! Let's find out!

Questions

Like all good experiments, this project begins with a few questions:

  • Can I read my own brain data?
  • If so, is the data valid?
  • Can I improve my performance in video games by being super focused and concentrating?
  • If I'm distracted, will I play poorly?
    • Will a "bad game" be visible in my brain activity?
  • Is there a direct measurable relationship between superior gaming performance and high levels of attention?
    • Or, are there just too many factors at play?

Goals & Constraints

  • Capture brain data (at home) in an accurate, and cost-effective manner
  • Establish a "baseline" mental state
  • Observe the difference between the baseline state and the "game playing" state
  • Observe my brain's "relaxation" state while playing the game
  • Quantify video game performance (via stats)
  • Establish and visualize a relationship between my cognitive measures and my game performance
  • Can outside factors be excluded?
    • Server/network latency
    • "Skill based match making"
    • Other unknowns

Hypothesis

The more focused I am, the better I will perform at a multiplayer "first-person shooter" (Call of Duty: Vanguard). Conversely, when I'm distracted, my performance will be negatively impacted.

Detailed Approach

  • Using a "mind flex" toy, brain activity will be captured by "hacking" the serial output of the device to collect the EEG related information from the toy.
  • Brain activity will be uploaded to the cloud
  • Separately, video game stats will be captured via a scheduled task in a cloud based application
  • Brain and video game data will be combined and analyzed

Testing Steps

  • Hack "Mind Flex" toy by inserting ESP/Arduino to read serial data
    • Produce brain data to MQTT endpoint (`bttc/brain`)
      • Capture "distractions" via button
    • Produce brain data to "live demo" endpoint (to avoid persisting conference volunteer data) (`bttc/demo`)
  • Create serverless function to obtain video game stats
    • Use third-party API to retrieve match info
  • Create server side application
    • Consume MQTT endpoints
      • Persist brain data to cloud DB
      • Persist distractions to cloud DB
    • Schedule task to invoke serverless function
      • Persist match information to cloud DB
    • Create a user interface (UI) to demo "live" brain data
      • Chart metrics in near real-time
    • Create a user interface (UI) to compare persisted brain data with persisted video game stats
  • Collect data
    • Play game while wearing "mind flex"
      • Play without distractions
      • Play with distractions
